## **Creating a User in Tomcat & Changing the Default Port Number**

### **Why Change Tomcat Port Number?**

✦ Apache Tomcat uses port `8080` by default.

✦ Sometimes another application may already be using port `8080`.

✦ In that case, Tomcat cannot start properly.

✦ So we change the Tomcat port number to another available port such as:

* `9090`
* `8081`
* `7070`

### **Example**

```text
Tomcat              → 8080
Another Application → 8080
```

Both applications cannot use the same port at the same time.

So we change the Tomcat port:

```text
Tomcat → 9090
```

### **Short Note**

✦ Tomcat port number is changed when port `8080` is already being used by another application or when a different port is required.

## **User**

### **Why Do We Create User Accounts in Tomcat?**

✦ User accounts in Apache Tomcat are created for authentication and security.

✦ They are mainly used to access the Tomcat Manager Application and Admin Console.

### **Why Needed?**

Without a user account:

```text
Anyone could access Tomcat management features.
```

That is a security risk.

So Tomcat requires:

* Username
* Password
* Roles / Permissions

### **What Can We Do After Login?**

Using the Tomcat Manager GUI, users can:

* Deploy applications (`.war` files)
* Start applications
* Stop applications
* Restart applications
* Monitor running applications

### **Where Are Users Created?**

Inside:

```text
tomcat-users.xml
```

**File Location:**

```text
Tomcat/conf/tomcat-users.xml
```

### **Example**

```xml
<user username="admin" password="admin123" roles="manager-gui"/>
```

### **Short Note**

✦ User accounts are created in Tomcat to securely access the Tomcat Manager and Admin applications for deployment and server management tasks.

## **Tomcat Configuration Files**

✦ To change the Tomcat port number, use the `server.xml` file.

✦ To create users and manage roles/permissions, use the `tomcat-users.xml` file.

### **File Locations**

```text
Tomcat/conf/server.xml
Tomcat/conf/tomcat-users.xml
```

## **Changing the Port Number**

### **Edit server.xml**

1. Open the Tomcat folder:

```bash
cd TOMCAT_FOLDER_NAME
```

2. Enter the `conf` directory:

```bash
cd conf/
```

3. Edit the `server.xml` file:

```bash
sudo vi server.xml
```

4. Scroll down until you find:

```xml
<Connector port="8080" protocol="HTTP/1.1"/>
```

5. Press **Insert** and change the port number from `8080` to `9090` (or any available port).

**Example:**

```xml
<Connector port="9090" protocol="HTTP/1.1"/>
```

6. Save and exit:

```text
Esc
:wq!
```

---

### **Restart Tomcat**

```bash
shutdown.sh
startup.sh
```

---

### **Allow Port in Security Group**

1. Open the EC2 Instance.
2. Click **Security**.
3. Click **Security Groups**.
4. Click **Edit Inbound Rules**.
5. Click **Add Rule**.

Enter:

```text
Port Number : 9090
Source      : Anywhere-IPv4
```

6. Save the rule.

---

### **Verify in Browser**

1. Copy the Public IP Address of the EC2 Instance.
2. Open a new browser tab.
3. Enter:

```text
13.232.183.75:9090
```

If the Tomcat page appears, the configuration is successful.

## **Creating a User**

### **Edit tomcat-users.xml**

1. Open the Tomcat folder:

```bash
cd TOMCAT_FOLDER_NAME
```

2. Enter the `conf` directory:

```bash
cd conf/
```

3. Edit the file:

```bash
sudo vi tomcat-users.xml
```

4. Scroll down until:

```xml
</tomcat-users>
```

5. Press **Insert** and add the following lines above `</tomcat-users>`:

```xml
<role rolename="manager-gui"/>
<role rolename="manager-script"/>
<user username="admin" password="admin" roles="manager-gui,manager-script"/>
```

The final section should look like:

```xml
<role rolename="manager-gui"/>
<role rolename="manager-script"/>
<user username="admin" password="admin" roles="manager-gui,manager-script"/>

</tomcat-users>
```

6. Save and exit:

```text
Esc
:wq!
```

---

### **Restart Tomcat**

```bash
shutdown.sh
startup.sh
```

---

### **Verify Access**

1. Ensure port `9090` is allowed in the Security Group.
2. Open:

```text
http://13.232.183.75:9090
```

3. Click:

```text
Manager App
```

4. Enter:

```text
Username : admin
Password : admin
```

5. Login successful.

---

### **Short Note**

✦ `server.xml` is used to change the Tomcat port number.

✦ `tomcat-users.xml` is used to create users and assign roles.

✦ After modifying either file, Tomcat must be restarted for the changes to take effect.
